## 🚀 新的工作流程

### 开发流程
```bash
# 1. 本地开发和预览
npm run server
# 访问 http://localhost:4000

# 2. 写文章
hexo new post "文章标题"

# 3. 提交源码
git add source/_posts/新文章.md
git commit -m "添加新文章：xxx"
git push origin master
```

### 部署流程
```bash
# 1. 清理旧构建
npm run clean

# 2. 生产构建（包含压缩优化）
npm run build:prod

# 3. 部署到 GitHub Pages
npm run deploy
```

## ⚠️ 需要手动处理的问题

### 1. 依赖包更新（可选）
```bash
# 保守更新
npm update

# 或大版本升级（需要测试）
npm install hexo@latest --save
```

**主要可更新的包：**
- hexo: 5.4.2 → 8.1.2
- gulp: 4.0.2 → 5.0.1
- 多个 hexo 插件

### 2. 移除冗余 Markdown 渲染器
```bash
# 推荐保留 markdown-it（功能更强）
npm uninstall hexo-renderer-marked

# 或保留 marked（更轻量）
npm uninstall hexo-renderer-markdown-it
```

### 3. 安全漏洞修复（重要！）
**问题**：Gitalk OAuth clientSecret 暴露在配置文件中

**位置**：`themes/hexo-theme-matery/_config.yml:342-343`

**解决方案**：
1. 立即前往 GitHub Settings → Developer settings → OAuth Apps
2. 重新生成 Client Secret
3. 考虑切换到更安全的评论系统（如 Utterances）

### 4. 仓库架构优化（建议）
当前配置：源码和部署都在同一个仓库的 master 分支

**建议改进**：
```yaml
# _config.yml
deploy:
  type: git
  repo: https://github.com/Parade001/Parade001.github.io.git
  branch: gh-pages  # 改为 gh-pages 分支
```

然后在 GitHub 仓库设置中，将 Pages 源设置为 `gh-pages` 分支。
